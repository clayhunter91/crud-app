When deploying on Vercel, the command "npm run build" gave me an error: Command "npm run build" exited with 1.

I couldn't understand why, since I had the environment variables and files configured correctly. So, I thought to reproduce the error in the terminal using the same command, and there it indicated Module not found: Can't resolve '@/styles/globals.css'


I researched the issue and found out it was because there is a file named _app.js inside "pages" that contains an import with an alias '@' on the first line.


Then I corrected it by adapting the next.js to be able to use aliases when referencing directories. If you modify next.js with this change, it can be deployed without problems. Here are the changes introduced in the file 1014-todo-app-fastapi-vercel-fullstack/002-nextjs-frontend/todo-app/next.config.js:


// Import the 'path' module to resolve paths

const path = require('path');


const nextConfig = {

  reactStrictMode: true,


  // Extend the Webpack configuration

  webpack: (config) => {

    // Set up an alias '@' that points to the root directory of the project

    config.resolve.alias['@'] = path.resolve(__dirname);

    return config;

  },

}


module.exports = nextConfig;
